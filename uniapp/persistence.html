    持久化

    # 持久化

    当使用 vuex 或 pinia 时，默认情况下全局状态值存储在内存中。然而，这意味着在应用重新打开时数据会丢失。如果希望某些全局数据在下次打开应用时仍然可用，例如登录凭证，这就需要进行持久化处理。

    常见的持久化方式：

    • 使用浏览器的本地存储
    • 使用数据库
    • 使用其他持久化存储方案，如 IndexedDB、WebSQL 等

    在这里，我们主要使用 uniapp 提供的 setStorageSync 方法来进行数据存储。

    # 封装基础存储工具

    // store.js

export const get = (key) => {
  return uni.getStorageSync(key);
};
export const set = (key, value) => {
  uni.setStorageSync(key, value);
};
export const remove = (key) => {
  uni.removeStorageSync(key);
};
export const clear = () => {
  uni.clearStorageSync();
};

    这样设计的好处，后面替换底层的存储方案只需要改这里。 同样添加统一的检查逻辑，比如数据序列号、反序列化。

    # 封装存储 token 存储工具

    // utils/token.js

import { get, set, remove } from './store.js';
const tokenPrefix = 'token';
export const setToken = (token) => {
  set(tokenPrefix, token);
};
export const getToken = () => {
  return get(tokenPrefix);
};
export const hasToken = () => {
  const token = getToken();
  return !!token;
};
export const removeToken = () => {
  remove(tokenPrefix);
};

    # 持久化时机

    vuex 或 pinia 提供了扩展选项，允许外部以插件的形式订阅全局数据操作，例如 mutation 和 action。通过这种方式，用户可以选择性地对数据进行持久化，具体取决于操作的名称。

    通过这种订阅方式，可以使简化代码，让代码变得更加整洁和可维护、 与业务逻辑解耦、提高代码的复用性。

    // stores/localstoragePlugin.js
import { setToken } from '@/utils/token';
export const localstoragePlugin = ({ store }) => {
  store.$onAction((action) => {
    if (action.name === 'setToken') {
      // 存储 token
      setToken(action.args[0]);
    }
    // ... 依次类推
  });
};

// 使用插件
import { localstoragePlugin } from '@/stores/localstoragePlugin.js';
const pinia = Pinia.createPinia();
pinia.use(localstoragePlugin);

    # 将 store 数据回填

    在应用重新打开时，需要将之前存储在本地的数据重新填充到 vuex 或 pinia 的state中。

    通常情况下，数据回填只需要执行一次。请思考一下，uniapp 是否提供了相应的钩子函数来方便我们完成这个操作。

    <script>
import { useUserStore } from '@/stores/user.js';
export default {
  onLaunch: function () {
    // 应用启动时，填充全局变量
    console.log('App Launch');
    // 回填数据
  },
  onShow: function () {
    console.log('App Show');
  },
  onHide: function () {
    console.log('App Hide');
  },
};
</script>
