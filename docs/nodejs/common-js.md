# Node.js 中的 Common JS 规范实现

## 1. Node 模块的实现
在 Node 中引入模块，需要经历如下3个步骤：

* 路径分析
* 文件定位
* 编译执行

在 Node 中，模块分为两类：一类是 Node 提供的模块，称为``核心模块``；另一类是用户编写的模块，称为``文件模块``。

* 核心模块部分在 Node 源代码的编译过程中，编译进了二进制执行文件。在 Node 进程启动时，部分核心模块就被直接加载进内存中，所以这部分核心模块引入时，文件定位和编译执行这两个步骤可以省略掉，并且在路径分析中优先判断，所以它的加载速度是最快的。
* 文件模块则是在运行时动态加载，需要完整的路径分析、文件定位、编译执行过程，速度比核心模块慢。

## 2. 文件查找和定位

### 2.1 优先从缓存加载

与浏览器缓存静态脚本文件以提高性能一样，``Node`` 对于加载过的内容会进行缓存，减少二次引入的开销。 不同的地方在于，浏览器缓存是文件，而 Node 缓存的是编译和执行之后的对象。 

```js
// a.js 先执行
import commonModule from ''commonModule
commonModule.cacheName = 'cacheName'

//b.js 后执行
import commonModule from ''commonModule
console.log(commonModule.cacheName); // cacheName
```

### 2.2 路径分析和文件定位

``require`` 接收一个``模块标识符``作为参数。在 Node 实现中，正是基于这样一个标识符进行模块查找。

标识符分类：

* 非路径形式：核心模块(内置模块)、第三方的依赖包
* 路径形式：``.`` 、``../``  或 ``/`` 开头的路径

#### 2.2.1 模块加载优先级

* 核心模块：优先级仅次于缓存加载，它在 Node 的源代码编译过程中已经编译为二进制代码，其加载过程最快。如果尝试去加载一个与核心模块相同的标识符，是不能被正常加载的 (加载优先级的问题)。
* 文件模块：是 ``.`` 或 ``../`` 以及 ``/`` 开始的标识符，都会按文件模块来处理。在分析路径模块时，``require`` 方法会将路径转为真实的路径，并已真实的路径作为索引，将编译执行后的结果存放到缓存中，以使二次加载时更快。


对于 ``非路径形式的标识符``，它的查找顺序是按 ``module.paths`` 路径数组进行查找， 其次，在引入模块时通常不会添加扩展名，例如：

```
require('lodash')
```

那么 Node 在查找时，也是做一些扩展名分析策略：

* 扩展名分析，优先按 ``.js`` 、``.json``、``.node`` 去查找
* 如果都没有查找到，则会查找当前目录下 ``package.json`` ，通过 ``JSON.parse`` 解析包描述对象，从中取出 ``main`` 属性指定的文件名及性能定位，如果缺扩展名，走扩展名分析。
* 如果不存在 ``package.json`` 或指定 ``main`` 无效，``Node`` 会将 ``index`` 当作默认文件名，然后依次查找 ``.js`` 、``.json``、``.node``
* 如果当前目录没有查找到，会依次按 ``module.paths`` 一层一层的往上查找，重复 1-3步骤，依然未找到，则会报未到找对应模块异常。

对于 ``路径形式标识符``，它的查找方式与非路径形式有所区别，只从当前模块的文件所在目录开始解析，不会向上搜索 ``node_modules`` 路径:

* 扩展名分析，优先按 ``.js`` 、``.json``、``.node`` 去查找
* 如果都没有查找到，则会查找当前目录下 ``package.json`` ，通过 ``JSON.parse`` 解析包描述对象，从中取出 ``main`` 属性指定的文件名及性能定位，如果缺扩展名，走扩展名分析。
* 如果不存在 ``package.json`` 或指定 ``main`` 无效，``Node`` 会将 ``index`` 当作默认文件名，然后依次查找 ``.js`` 、``.json``、``.node``

举一个例子：

```js
require("./test")
```

* 解析步骤（类似非路径，但范围限于当前目录）：
    * 尝试 ./test 作为文件，补扩展名：test.js → test.json → test.node。
    * 如果是目录，检查 ./test/package.json 的 main 字段（补扩展名）。
    * 如果无效，回退到 ./test/index.js → ./test/index.json → ./test/index.node。

如果当前目录下找不到，会抛 ``MODULE_NOT_FOUND`` 错误，不会“爬”到上级。
