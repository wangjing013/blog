# JSX

``JSX`` 是 ``JavaScript`` 的一种语法扩展，它允许你在 ``JavaScript`` 文件中编写类 ``HTML`` 的标签元素。尽管还有其他编写组件的方法，大多数``React``开发者喜欢``JSX``简单的语法并且大多数代码库都在使用它。

* 为什么 React 将标签与渲染逻辑混合在一起
* JSX 与 HTML 之间的不同
* JSX 规范

## 标签与逻辑混在一起的原因

多年以来，编写代码习惯都是把网页内容存在 HTML 文件中，样式放在 CSS 文件，逻辑则放在 JavaScript 中 -- 通常是在不同的文件中！

但随着 ``Web`` 的交互性越来越强，逻辑越来越决定页面中的内容。

``JavaScript`` 控制着 ``HTML`` 的内容！这也是为什么在 ``React`` 中，渲染逻辑和标签共同存在于同一个地方——组件。

## JSX 与 HTML 的不同

``JSX`` 虽然看起来很像 ``HTML``，但在底层其实被转化为了 ``JavaScript`` 对象，你不能在一个函数中返回多个对象，除非用一个数组把他们包装起来。这就是为什么多个 ``JSX`` 标签必须要用一个父元素或者 ``Fragment`` 来包裹。


## JSX 规范
* 只能返回一个根元
* 标签必须是闭合的
* 使用驼峰命名法给大部分属性命名
    * JS 对变量的命名有限制，比如是字母数据下划线，且不能使用像 ``class`` 这样的保留字，这就是为什么会使用 ``className`` 来表示 ``class``。其他属性类推。


## 总结

* 逻辑与视图紧密相关，所以 React 将它们放在一个组件中
* JSX 类似 HTML，但是 JSX 最终会转化为 JavaScript 对象
* JSX 属性命名采用驼峰的原因
* 使用[转化器](https://transform.tools/html-to-jsx)将 HTML 转换为 JSX
